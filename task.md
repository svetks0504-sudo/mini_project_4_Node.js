1.	Проверка email: Когда пользователь пытается войти в систему, мы сначала проверяем, существует ли пользователь с таким email в базе данных.

3.	Генерация JWT: Если пароли совпадают, мы создаем JWT, который будет содержать идентификатор пользователя. Этот токен подписывается секретным ключом, который хранится на сервере.

4.	Возврат JWT: Токен отправляется клиенту, который затем будет использовать его для доступа к защищенным ресурсам.

Задание 6
Создание, редактирование и удаление задач
1.  Для работы с задачами создайте модель задачи (Task) с помощью Mongoose со следующими полями:
-   `title`: заголовок задачи, обязательное поле.
-   `description`: описание задачи, необязательное поле.
-   `status`: статус задачи, с возможными значениями "pending" (ожидание), "in progress" (в процессе) и "completed" (завершена). По умолчанию задача находится в статусе "pending".
-   `createdAt`: дата создания задачи, устанавливается автоматически при создании.
-   `user`: ID пользователя, создавшего задачу, обязательное поле, связывает задачу с конкретным пользователем.
2.  В файле `controllers/taskController.js` создайте контроллер для создания новой задачи.
3.  Добавьте маршрут для создания задачи в файле `routes/task.js`.
4.  Аналогичным образом добавьте контроллеры и маршруты для обновления и удаления задач.
Пользователи могут захотеть отфильтровать свои задачи по статусу (например, "в процессе", "выполнено") или дате создания (например, задачи, созданные сегодня, за последнюю неделю и т.д.).
-   Фильтрация по статусу: Мы можем добавить параметр запроса (`query parameter`), который позволяет пользователю указать статус задач, которые он хочет видеть (например, `?status=in progress`).
-   Фильтрация по дате: Также можно добавить параметры для фильтрации по дате создания задачи. Например, можно искать задачи, созданные после определённой даты (`?startDate=2024-09-01`) или до определённой даты (`?endDate=2024-09-04`).
Всё это мы будем передавать через параметры запроса (`query parameters`), чтобы гибко фильтровать задачи.
