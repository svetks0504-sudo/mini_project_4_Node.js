1.	Проверка email: Когда пользователь пытается войти в систему, мы сначала проверяем, существует ли пользователь с таким email в базе данных.

2.	Сравнение пароля: Если пользователь найден, мы сравниваем введенный пароль с хешированным паролем, который хранится в базе данных. Для этого используем метод `bcrypt.compare`.

3.	Генерация JWT: Если пароли совпадают, мы создаем JWT, который будет содержать идентификатор пользователя. Этот токен подписывается секретным ключом, который хранится на сервере.

4.	Возврат JWT: Токен отправляется клиенту, который затем будет использовать его для доступа к защищенным ресурсам.
